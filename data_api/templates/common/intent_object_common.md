# Intent Object 共通定義

このファイルは個別意図オブジェクトと上位意図オブジェクトに共通する概念定義を含みます。

## intent の定義

何を目指しているか、目的としているか、タスクなどに該当するもの

以下を両方含むこと:
- **志向対象（Target / Object）**: 何に向かっているのか（目的・目標・状態・価値など）
- **志向方向性（Directionality）**: 主体がその対象に対してどうしたいのか、「○○を達成したい」「△△を避けたい」など

| 内容 | intent の適切さ | コメント |
|------|-----------------|----------|
| 作業効率を上げたい | ○ | 志向対象＝「作業効率」、志向方向性＝「上げたい」。目的が明確で intent として適切。 |
| ユーザーが迷わず使えるUIを実現したい | ○ | 志向対象＝「ユーザーが迷わず使えるUI」、志向方向性＝「実現したい」。具体的で方向性も明確。 |
| バグを減らす | ○ | 志向対象＝「バグ」、志向方向性＝「減らす」。改善志向の明確な intent。 |
| チームのモチベーション向上に取り組む | ○ | 志向対象＝「チームのモチベーション」、志向方向性＝「向上させる／取り組む」。行動的で目的が明確。 |
| より良くしたい | △ | 「良くしたい」という方向性はあるが、何を（対象）が不明。志向対象が欠けている。 |
| 新しいことに挑戦 | △ | 対象＝「新しいこと」は曖昧。方向性は「挑戦する」としてある程度示されている。 |
| デザインが気になる | ✕ | 感情・印象の表明であり、「どうしたいか（方向性）」が欠けている。 |
| 時間が足りない | ✕ | 状況の記述であり、目的や方向性が含まれない。 |
| ミスを減らせたらいいな | △ | 対象＝「ミス」、方向性＝「減らしたい」だが、推量・願望的で弱い。曖昧な意図。 |
| ユーザーの離脱を防ぎたい | ○ | 志向対象＝「ユーザーの離脱」、志向方向性＝「防ぎたい」。負の方向を避ける意図として明確。 |

## status の定義

- **idea**: まだ具体的な行動には至っていないアイデア・構想段階
- **todo**: やるべきこととして認識されているが、まだ着手していない
- **doing**: 現在進行中、作業中
- **done**: 完了している

※「意図はあるが対象または方向性が不明瞭な場合」は一律で "idea" として扱う。

## objective_facts の定義

主観・印象・推測・評価・感情・価値判断を完全に排除し、観察可能な出来事・状態・数量・時間・場所・行動だけを抽出する。

🔹 LLM が判断するときの基準（必ずすべて満たす）

1. 観察・記録された出来事か？
	•	日付・時間・距離・場所・回数・料金・天候・路線名など
	•	「〜した」「〜があった」「〜に乗った」のように観測可能な事実

→ YES なら objective_facts に入れてよい。

2. 他人が読んでも同じ事実と認識できるか？
	•	主観表現を含まないか
	•	「多い」「少ない」「遅い」「快適」「最適」などの評価語が含まれないか

→ 評価語がある場合は 事実ではない。

3. 内面状態（感情・思考・推測）が混ざっていないか？

以下は すべて禁止：
	•	感じた／思った／かもしれない
	•	最適／面白い／単調／旅感がない
	•	〜すべき
	•	〜したい（願望）
	•	〜と感じている（感情）

→ これらが入っていたら 客観的事実ではない。

| 内容 | 客観的事実の度合い | コメント |
|------|-----------------|----------|
| 作業記録やタスクが様々な場所に散逸している | △（部分的に事実） | 「散逸している」は観察に基づくが、どの範囲・どの媒体にかは明示されていない。 |
| 全体像が見えていない | ✕（主観） | 「見えていない」は認識や感情による表現。客観的ではない。 |
| 事実と感情は活用しやすい | ✕（主観） | 「活用しやすい」は主観的評価。 |
| 解釈の明確化が課題 | △ | 「課題」として挙げているが、観察に基づくなら事実的要素もある。 |
| 細かすぎる分類より4分類を先に目指すべきか葛藤している | ✕ | 「葛藤」は感情や思考。客観的ではない。 |
| 過去のメモアプリが単なるメモ垂れ流しになってしまった経験がある | ○ | 「過去にメモアプリを使っていて、整理できなかった」という経験は事実として扱える。 |
| その経験がアプリ開発の初期動機になっている | ○ | 原因と動機の説明として事実的に表現できる。 |
| 鉄道旅は思考を深めるのに最適かもしれない (msg_00123) | ✕ | 元のメッセージから引用するれば必ず事実である訳ではない。客観的現象に関する言及のみを抽出すること |
|「乗換案内から両方のチケット、ルートイン小諸、予約は必要 - task - 未完了」と明記されている。 | ✕ | 事実を抽出して。「明記されているから」客観的事実ではない
