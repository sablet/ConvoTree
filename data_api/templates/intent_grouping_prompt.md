# 意図のグループ化

## タスク概要

以下の意図リストを分析し、**共通のテーマ・目的を持つ意図をグループ化**してください。

あなたの役割は意図の内容を理解し、どの意図とどの意図が同じグループに属するかを判断することです。

## グループ化の基準

{COMMON_INTENT_OBJECT}

**重要**: グループを代表する名前（group_name）は、上記のintent定義と同様に、志向対象と志向方向性を必ず含むこと。

## グループ化ルール

1. **テーマの共通性**: 共通のテーマ・目的・領域を持つ意図をグループ化
2. **単一テーマの原則**: 1つのグループは1つの核心的なテーマに絞る。複数の異なるテーマは分割する
3. **プロジェクト文脈の活用**: source_full_pathsが同一プロジェクトならプロジェクト名を含める。複数プロジェクトで共通テーマなら汎用的表現で統合する
4. **排他的**: 1つの意図は1つのグループにのみ属する
5. **網羅的（必須）**: 全ての意図が必ずいずれかのグループに含まれること
6. **簡潔性**: group_nameは最大30文字以内

## 出力形式

グループごとに以下の形式で出力:

```json
{{
  "group_name": "グループを代表する意図（志向対象+志向方向性、30文字以内）",
  "objective_facts": "このグループ化を支える客観的事実（簡潔）",
  "context": "group_nameを理解するための背景・補足（簡潔）",
  "member_indices": [0, 2, 5]
}}
```

## 出力制約

### 禁止表現
- ❌ 「○○と△△と□□を××したい」（複数概念の並列列挙）
- ❌ 「○○における△△とAI連携による□□の実現」（過度な修飾）
- ❌ 「高度な」「包括的な」「効率的な」（抽象的修飾語の濫用）

### 推奨パターン
- ✅ プロジェクト特化: 「[プロジェクト名]の[課題]を[方向性]したい」
- ✅ プロジェクト横断: 「[汎用的活動]の[課題]を[方向性]したい」

### 重要事項
- 出力は純粋なJSON配列のみ（説明文・コメント禁止）
- 全ての意図（0〜{max_index}）がいずれかのグループに含まれること
- 1つの意図が複数のグループに属してはならない

例:
```json
[
  {{
    "group_name": "...",
    "objective_facts": "...",
    "context": "...",
    "member_indices": [0, 1, 3]
  }},
  {{
    "group_name": "...",
    "objective_facts": "...",
    "context": "...",
    "member_indices": [2]
  }}
]
```
