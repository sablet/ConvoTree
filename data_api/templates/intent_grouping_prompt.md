# 意図のグループ化

## タスク概要

以下の意図リストを分析し、**共通のテーマ・目的を持つ意図をグループ化**してください。

あなたの役割は意図の内容を理解し、どの意図とどの意図が同じグループに属するかを判断することです。インデックス管理やデータ構造の操作は不要です。

## グループ化の基準

{COMMON_INTENT_OBJECT}

**重要**: グループを代表する名前は、上記のintent定義と同様に、志向対象と志向方向性を必ず含むこと。

**良い例**:
- 「タスク管理システムの使いやすさを向上させたい」
- 「開発プロセスの効率を高めたい」
- 「個人の生産性と意思決定の質を向上させたい」

**悪い例**:
- 「色々な改善」（志向対象が曖昧）
- 「システム開発」（志向方向性が欠けている）
- 「改善する」（志向対象が欠けている）

## グループ化ルール

1. **テーマの共通性**: 共通のテーマ・目的・領域を持つ意図をグループ化
2. **抽象度の適切性**: 過度に抽象的（「改善全般」）でも、過度に具体的でもない中間レベルを目指す
3. **最小1つ以上**: 1つのグループには最低1つ以上の意図が含まれること
4. **重複なし**: 1つの意図は1つのグループにのみ属する（排他的）
5. **網羅性（必須）**: **全ての意図が必ずいずれかのグループに含まれること**
6. **単独意図の扱い**: どのグループにも属さない単独の意図は、それ自体を1つのグループとして扱う

## 出力形式

グループごとに以下の形式で出力:

```json
{{
  "group_name": "グループを代表する意図（志向対象+志向方向性を含む）",
  "description": "このグループの詳細説明",
  "rationale": "なぜこれらをグループ化したか",
  "member_indices": [0, 2, 5]  // このグループに属する意図のインデックス（0始まり）
}}
```

**重要**:
- 出力は純粋なJSON配列のみ（説明文・コメント禁止）
- 全ての意図（0〜{max_index}）がいずれかのグループに含まれること
- 1つの意図が複数のグループに属してはならない

例:
```json
[
  {{
    "group_name": "...",
    "description": "...",
    "rationale": "...",
    "member_indices": [0, 1, 3]
  }},
  {{
    "group_name": "...",
    "description": "...",
    "rationale": "...",
    "member_indices": [2]
  }},
  {{
    "group_name": "...",
    "description": "...",
    "rationale": "...",
    "member_indices": [4, 5]
  }}
]
```
