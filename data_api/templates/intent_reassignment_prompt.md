# 意図の再割り振りタスク

以下の意図グループ化において、一部の項目が既存のグループに割り当てられませんでした。
これらの未割り当て項目を、既存のグループに追加するか、または新しいグループを作成して割り当ててください。

## タスク

上記の未割り当て項目を、以下のいずれかの方法で処理してください：

1. **既存のグループに追加**: 未割り当て項目が既存のグループと関連性が高い場合、そのグループに追加
2. **新しいグループを作成**: 未割り当て項目が既存のどのグループにも合わない場合、新しいグループを作成

## 出力形式

以下のJSON配列形式で出力してください：

```json
[
  {{
    "group_name": "グループの説明",
    "objective_facts": "客観的な事実",
    "context": "背景情報",
    "member_indices": [インデックス1, インデックス2, ...]
  }}
]
```

**重要な制約**:
- `member_indices` は、**未割り当て項目のインデックス（0から{max_index}）のみ**を含めること
- 既存のグループに追加する場合も、追加する項目のインデックスのみを記載
- 新しいグループを作成する場合も、インデックスのみを記載
- **全ての未割り当て項目（インデックス0から{max_index}）を必ずいずれかのグループに含めること**
- 重複は許可されません（1つの項目が複数のグループに属することはできません）

## 出力例

```json
[
  {{
    "group_name": "既存グループAに追加",
    "objective_facts": "これらの項目はグループAと同じテーマに関連",
    "context": "グループAに追加すべき項目",
    "member_indices": [0, 2, 5]
  }},
  {{
    "group_name": "新しいグループB",
    "objective_facts": "これらの項目は独自のテーマを持つ",
    "context": "新規グループとして作成",
    "member_indices": [1, 3, 4]
  }}
]
```
